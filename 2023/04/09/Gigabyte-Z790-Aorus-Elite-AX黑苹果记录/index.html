Gigabyte-Z790-Aorus-Elite-AX黑苹果记录

发表于2023-04-09|更新于2023-04-09
|字数总计:728|阅读时长:2分钟|阅读量:

我的硬件

  • 主板：Gigabyte Z790 Aorus Elite ax
  • CPU：I7 13700K （对CPU没特别要求，别的CPU只需在config.plist中更改显示名就可以）
  • GPU：Gigabyte RX6600 EAGLE 8G (只要是免驱的显卡都应该都适用)
  • macOS：Ventura 13.3.1（Ventura所有版本都适用，Monterey、Big Sur没测试过）

EFI制作过程

按照官方教程OpenCore-Install-Guide/操作就行。

大致可分为

这些流程基本都很简单，我只记录自己当初卡壳的流程
  • 制作ACPI：是使用SSDTTime工具在Windows下生成的，操作很简单，只是当初看英文文档的时候先看的手动制作，以为很难，其实用工具在Windows下选择几个选项就自动生成了。
  • 制作macOS启动盘：我之前用的笔记本用别人的EFI做了黑苹果，就在macOS下用etcher制作黑果启动盘，但这个etcher工具有一个很大的坑就是EFI分区没标记，引导时不正常，我用它制作启动盘时切换了多个EFI引导都是找不到启动Mac的选项，一度以为全是EFI不配置我的主板。直到在远景上看到别人讨论才知道是这原因。如果你网速快的话用官方的方式制作在线恢复盘rufus-method安装就行。想离线安装就用分区工具把EFI分区标记成活动引导分区之类的（自己没验证过）。

BIOS设定

  • Secure Boot : Disabled
  • Internal Graphics : Disabled
  • Above 4G Decoding : Enabled
  • Above 4GB MMIO BIOS assignment : Enabled
  • Re-Size BAR Support : Disabled
  • Hyper-Threading : Enabled
  • CFG Lock : Disabled

其实就一个是最重要的，Secure Boot : Disabled，别的都不太重要。

Hyper-Threading：打开是超线程技术，只是Opencore支持的还不太好，还要对自己的CPU做驱动才支持，而CPUFriend之类的工具还对13代CPU支持的不太好。

CFG Lock：可以在Opencore里设置启动和关闭，不是必需在BIOS里设定。

Internal Graphics：核显开启也没关系，没影响

4G Decoding之类的：看别人贴子是影响睡眠唤醒之类的，我不玩电脑都是直接关机，没用唤醒睡眠功能，这个不影响我。再说这个看别人的说法还跟BIOS版本有关。

增量更新

禁用BlueToolFixup.kext 再检查更新安装即可增量更新

我是把蓝牙相关的三个kext（BlueToolFixup.kext、IntelBluetoothFirmware.kext、IntelBTPatcher.kext）都禁用增量更新成功了。

安装过程

安装过程中启动Apple Logo后会有一段较长时间的屏幕熄灭，我第一次安装的时候有好几分钟的黑屏时间，目前正常使用时启动有几秒的黑屏时间，黑屏时间过后才会出现设置界面或登录界面。

推荐的软件

  • MonitorControl ：用来调整屏幕亮度。
  • OCAT ：用来修改EFI文件
