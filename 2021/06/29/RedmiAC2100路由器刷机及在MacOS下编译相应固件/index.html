RedmiAC2100路由器刷机及在MacOS下编译相应固件

发表于2021-06-29|更新于2021-06-30|Linux
|字数总计:1.2k|阅读时长:3分钟|阅读量:

​ 前言：以前想用Github都得在电脑上操作，手机跟平板想用Github那是各种障碍，很不方便。今年五月份得知有Openwrt这么个东西就一直盼着618路由器能降价买个能刷机的来整。看高端的路由没一个降价的就买了红米AC2100凑合着用。到手检测是东芝的闪存有三个坏块，买电子产品都是提前做好功课的呵，嘿嘿，直接找客服换货才发来一个ESMT闪存没坏块的。

刷机教程网上一大堆，我用的是：**红米(小米)AC2100无需Telnet刷入Breed和Padavan固件教程。跟着教程刷入了Breed，就像Android手机的第三方Recovery**，刷了Breed以后再刷固件就不会变砖了。当时是在MacOS开虚拟机才刷的Breed，后来才发现连上SSH用终端一样的操作。

输入上面的命令后回车再输入密码就能连路由器了。

以上代码是复制本地文件1到路由器tmp目录下并保存为文件名2。scp命令是在没连ssh状态下操作的。

Openwrt官方固件已经支持RedmiAC2100(架构ramips、处理器mt7621)：https://downloads.openwrt.org/

官方固件没有Luci，也就是刷好后不能通过网页输入192.168.1.1来操作路由器，要用别的路由拨号正常上网，AC2100用网线连接介于正常路由与电脑之间后，再用ssh连接ac2100通过opkg update && opkg install luci来安装Luci。官方固件里功能极少，极为精简，使用时发热也很少，但少了我必要的几个功能。就琢磨着自己编译固件。可是我只有Mac环境，配置也不怎么样。看网上有使用Github的Actions来云编译固件的（Actions-OpenWrt），编了快十次，只有两次成功了，而且想要的插件也没能成功附上。就只能本地编译。

虚拟机里开Ubuntu，使用了半小时，系统实在难用，改用Manjaro。在Manjaro里clone代码编译，总是在make menuconfig后再make的时候也跳出来选配置的对话框，还以为是哪里配置不对，代码删了又下，下了又删，官方源码、三方源码换来换去都编不过，好不容易有一次开头过了，编译两三个小时后还是失败，又没错误输出，加上又是虚拟机里搞得，懒得折腾。目前网上的教程都是推荐在Ubuntu里编译，官网也没把MacOS放在支持的列表中。但自己真的有点信仰这系统，决定试试，换到MacOS环境编译(11.4)，使用磁盘工具创建一个区分大小写的虚拟盘，再clone代码，执行./scripts/feeds update -a后会在终端提示配置环境缺少哪些工具，我目前环境提示少了四个，使用brew search搜索这四个工具再安装就行了。还有一点是make工具换成GNU的make。

报错一：

然后我是跟着官网提示操作的./scripts/feeds update -a、./scripts/feeds install -a、make menuconfig、make，到最后一步make的时候死活不行，还是一样的情况弹选配置选择的对话框，可在make menuconfig那一步已经选过了。删代码，下官网Openwrt源码，不再下插件，直接编译，也报错，立马怀疑人生了，，再一想官方源码不会有错的，一定我哪里操作出错了。报错跟原来的不一样就搜索一下报错内容：

找到一个说是依赖的东西没成功下载。试了一下果然。看来真的是应了之前看到一位网友的话，编译出错，百分之九十九都是跟网络连接有关的。这下真信了。

在最后一步之前执行make download V=s等待下载完依赖，再执行make就能成功编译了。

报错二：

编译了两个多小时了也没报错，以为成功了，高高兴兴写记录，结果现在error了。

使用make -j1 V=sc来编译并查看log发现提示说与VMware Fusion的环境变量冲突，期间还看到有人几年前用MacOS编译过，于是卸载了虚拟机再试着编译成功了。这次真的编译成功了。

报错三：

看了下生成的文件是默认的ath79架构，没有生成我需要的固件。不过已经说明MacOS可以正常编译了。明天再试吧，又折腾了一下午，唉。
